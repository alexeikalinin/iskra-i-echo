# Обновление HANDOFF.md (Искра и Эхо)

## Когда применять

- Завершил заметную задачу (багфикс, фича, рефакторинг с эффектом на флоу).
- Пользователь просит «записать в handoff», «не потерять контекст», «что сделано».
- Перед долгим перерывом или передачей работы другому агенту/разработчику.

## Где файл

`HANDOFF.md` в **корне проекта** (рядом с `CLAUDE.md`).

## Формат записи (обязательно)

Вставь **новый блок** сразу **перед** разделом `<!-- АВТОМАТИЧЕСКИЕ ЗАПИСИ НИЖЕ` (ручные записи — выше автологов хука).

Используй шаблон из шапки `HANDOFF.md`:

```
### [ГГГГ-ММ-ДД] Краткий заголовок сессии
**Статус:** что сделано и что работает
**В процессе:** что осталось (или «нет»)
**Следующий шаг:** конкретная следующая задача для следующей сессии
**Изменённые файлы:** перечень путей от корня репо
**Заметки:** риски, решения, ссылки на баги/идеи по желанию
```

Правила:

- Пиши по-русски, коротко, по делу.
- **Изменённые файлы** — реальные пути вида `js/scenes/ChapterScene.js`, без выдуманных файлов.
- Если менялось много однотипных сцен, допустимо: «все мини-сцены с …» + один пример.
- Не дублируй длинные diff’ы; суть изменений — в **Статус** и **Заметки**.

## Что не делать

- Не перезаписывай весь `HANDOFF.md` и не удаляй старые записи.
- Не редактируй блоки автологов хука ниже `<!-- АВТОМАТИЧЕСКИЕ` без запроса пользователя.

## Связь с проектом

В `CLAUDE.md` указано, что порядок скриптов и сцены критичны; в handoff при системных правках упоминай `index.html` / `main.js` / `service-worker.js`, если трогал.
