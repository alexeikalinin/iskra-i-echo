## Архитектура и системы

- 💡 **2026-04-15** — Система мультиагентной оркестрации: лид-агент распределяет задачи между специализированными агентами (разработчик мини-игр, нарратив-дизайнер, дизайнер уровней, визуальный разработчик, системный разработчик). Агенты работают автономно по документам Character Bible, MiniGames Specs, Story Breakdown.
- 💡 **2026-04-15** — Автоматический HANDOFF.md — при завершении сессии хук записывает состояние работы, чтобы следующая сессия начиналась с точного места.

## Этап 7: ChapterScene + Диалоговая система

- 💡 **2026-04-15** — **ChapterScene** — связующая сцена между выбором компаньона и цепочкой мини-игр. Показывает название главы, локацию, активных компаньонов, первый диалоговый бит и запускает первую мини-игру цепочки из `CHAPTER_MINI_GAMES`.
- 💡 **2026-04-15** — **DialogueManager** — синглтон для воспроизведения диалогов поверх любой сцены. Поддерживает: имя персонажа, текст, эмоцию, avatar-иконку. Данные берутся из Character Bible.
- 💡 **2026-04-15** — **Переход CompanionSelectScene → ChapterScene** — после выбора компаньона плавный fade-переход в ChapterScene с главой 1 («Первая искра»).

## Этап 8: Match-3

- 💡 **2026-04-15** — Match-3 реализовать первым — это самая частая мини-игра (встречается в главах 1, 2, 3, 4, 5, 7, 9 и далее). Сетка 5×5 для первых глав, до 8×8 в поздних.
- 💡 **2026-04-15** — Светля реагирует в реальном времени на комбо: при 3+ комбо подряд — ярче светится, искры летят (Particle System Phaser 3).
- 💡 **2026-04-15** — Элементы Match-3 тематические: «световые осколки», «звёздные пылинки», «капли памяти» — меняются по главам под локацию.

- 💡 **2026-04-15** — Для анимации компаньонов использовать программные текстуры (уже есть в PreloadScene) + Tween-цепочки. Реальные спрайты добавить позже через AI-арт по промптам из Character Bible.

## Монетизация и retention

- 💡 **2026-04-15** — Lives система: 1 попытка на мини-игру → retry стоит жизнь → восстанавливается со временем (30 мин/жизнь, макс 5 жизней). Стандартная casual-модель.
- 💡 **2026-04-15** — Battle Pass для сезонных событий (Зимний Фестиваль Света, Ночь Теней и др.) добавить в пост-гейм после реализации основных 15 глав.
